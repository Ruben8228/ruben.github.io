Unidad 1: Modelos de Arquitectura

  • 1.1 Modelos de arquitectura
    • 1.1.1 Clásicos: Los modelos clásicos de arquitectura se basan en el diseño de von Neumann, caracterizado por un almacenamiento compartido de datos e instrucciones.
    • 1.1.2 Segmentados: Este modelo divide el procesamiento en varias etapas, como la segmentación de instrucciones, para optimizar el rendimiento.
    • 1.1.3 De multiprocesamiento: Utilizan múltiples procesadores para ejecutar varias tareas simultáneamente, mejorando la eficiencia en sistemas complejos.
  • 1.2 Análisis de los componentes
    • 1.2.1 Arquitectura
      • 1.2.1.1 Unidad central de procesamiento: Es el cerebro del sistema, encargado de ejecutar instrucciones y coordinar operaciones.
      • 1.2.1.2 Unidad aritmética lógica: Realiza cálculos matemáticos y operaciones lógicas esenciales.
      • 1.2.1.3 Registros: Almacenes pequeños y rápidos dentro del procesador para datos temporales.
      • 1.2.1.4 Buses: Canales de comunicación para transferir datos entre componentes del sistema.
    • 1.2.2 Memoria
      • 1.2.2.1 Conceptos básicos del manejo de memoria: Incluyen la organización y acceso a datos en la memoria principal y secundaria.
      • 1.2.2.2 Memoria principal: Almacena datos e instrucciones utilizados activamente por el procesador.
      • 1.2.2.3 Memoria caché: Una memoria rápida que almacena datos frecuentemente utilizados para acelerar el acceso.
    • 1.2.3 Manejo de entrada/salida
      • 1.2.3.1 Módulos de entrada/salida: Facilitan la comunicación entre el sistema y dispositivos periféricos.
      • 1.2.3.2 Entrada y salida programada: Un método en el que el procesador controla directamente las operaciones de E/S.
      • 1.2.3.3 Entrada/salida mediante interrupciones: Mejora la eficiencia al permitir que el procesador responda solo cuando se le solicita.
      • 1.2.3.4 Acceso directo de memoria: Permite transferencias rápidas entre memoria y dispositivos sin intervención del procesador.
      • 1.2.3.5 Canales y procesadores de entrada/salida: Son controladores dedicados que manejan tareas de E/S de manera autónoma.
    • 1.2.4 Buses
      • 1.2.4.1 Tipos de buses: Incluyen buses de datos, de direcciones y de control.
      • 1.2.4.2 Estructura de buses: La organización física y lógica que conecta los componentes.
      • 1.2.4.3 Jerarquía de buses: Describe cómo los buses están organizados para optimizar el rendimiento.
    • 1.2.5 Interrupciones: Mecanismo que permite al procesador responder a eventos externos o internos de manera eficiente.

Unidad 2: Estructura y Funcionamiento de la Unidad Central de Procesamiento

  • 2.1 Organización del procesador: Describe cómo se interconectan los elementos internos para ejecutar instrucciones.
  • 2.2 Estructura de registros
    • 2.2.1 Registros visibles para el usuario: Utilizados para almacenar datos temporales durante la ejecución de programas.
    • 2.2.2 Registros de control y estados: Almacenan información sobre el estado del procesador y las operaciones en curso.
    • 2.2.3 Ejemplos CPU: Analiza la estructura y características de procesadores específicos.
  • 2.3 El ciclo de la instrucción
    • 2.3.1 Ciclo Fetch-Decode-Execute (FDE): Describe cómo el procesador obtiene, interpreta y ejecuta instrucciones.
    • 2.3.2 Segmentación de instrucciones: Divide el ciclo de instrucción en etapas para mejorar el rendimiento.
    • 2.3.3 Modos de direccionamiento: Técnicas para acceder a datos en memoria.
    • 2.3.4 Estudio y análisis de CPU: Evalúa el rendimiento y diseño de procesadores.

  • 3.1 Chipset
    • 3.1.1 Unidad central de procesamiento: Elemento clave para el rendimiento del sistema.
